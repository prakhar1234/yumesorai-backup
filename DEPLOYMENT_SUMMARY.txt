⚙️ HOW IT WORKS
════════════════════════════════════════════════════════════════════════

1. You make code changes
2. git add . && git commit -m "Your change" && git push
3. GitHub Actions workflow triggers automatically
4. App builds and tests automatically
5. App deploys to Railway automatically
6. Your live site updates with new code
7. Workflow notifies you of success/failure

No manual deployment steps needed after initial setup!


🔐 ENVIRONMENT VARIABLES
════════════════════════════════════════════════════════════════════════

Required for CI/CD to work:

RAILWAY_TOKEN
  ├─ Get from: https://railway.app/account/tokens
  ├─ Add to: GitHub Repo → Settings → Secrets → New Secret
  ├─ Name: RAILWAY_TOKEN
  └─ Value: (your Railway token)

Optional (add later if needed):

HUBSPOT_API_KEY
  └─ For CRM integration

RESEND_API_KEY
  └─ For email notifications

Add these in Railway Dashboard → Variables

🔄 ROLLBACK / REVERT
════════════════════════════════════════════════════════════════════════

If something breaks after deployment:

Via Railway Dashboard
  → Deployments tab
  → Click previous successful deployment
  → Click Redeploy

Via Git
  $ git revert HEAD
  $ git push origin main
